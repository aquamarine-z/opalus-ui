## Variants

### Custom Modal (modal.custom)

The `modal.custom` function allows you to create a fully customizable modal that returns a value when closed.

<ComponentExampleDisplayer componentName={"modal"} exampleName={"variant-custom"} client:load/>

#### Signature

```tsx
modal.custom: <T = any>(
  content: (close: (result?: T) => Promise<void>) => React.ReactNode
) => Promise<T | undefined>;
```
#### Parameters

**`content: (close: (result?: T) => Promise<void>) => React.ReactNode`**
A function that returns the React content of the modal.
It receives a `close` function that you can call to close the modal.
You can pass an optional result to `close`, which will resolve the returned promise.

#### Returns

**`Promise<T | undefined>`**
A promise that resolves with the value passed to `close`,
or `undefined` if the modal was closed automatically or without a value.

#### Notes

- You can use any React hooks (`useState`, `useEffect`, etc.) inside the modal content.
- You can return any type `T` (string, object, boolean, etc.) from `close`.
- The modal behaves like an async function — you can `await` it directly.
- The modal must return **Shadcn `DialogContent`** or a compatible component as its top-level element.



### Alert Modal (modal.alert)

The `modal.alert` function allows you to show a simple alert modal with a title, message, and a close button. It internally uses `modal.custom` and returns a `Promise` that resolves when the modal is closed.

<ComponentExampleDisplayer componentName={"modal"} exampleName={"variant-alert"} client:load/>

#### Signature

```tsx
export type ModalOptions = {
    hasCloseButton?: boolean;
    title?: ReactNode;
    titleIcon?: ReactNode;
    showTitleIcon?: boolean;
}
export type AlertModalOptions = {
    message?: ReactNode | ReactNode[];
    closeButtonContent?: ReactNode;
} & ModalOptions;
modal.alert: (options: AlertModalOptions) => Promise<null>;
```
#### Parameters

**`options: AlertModalOptions`**
An object with the following optional properties:

- `title?: string` – The title of the alert. Defaults to `"Alert"`.
- `message: ReactNode` – The main message content of the alert.
- `closeButtonContent?: ReactNode` – Custom content for the close button. Defaults to an X button at the right top.

#### Returns

**`Promise<null>`**
A promise that resolves when the alert modal is closed. It does not return any meaningful value.

#### Notes

- The alert modal is fully controlled by `modal.custom`, but you only need to provide the `AlertModalOptions`.
- The modal uses `DialogContent`, `DialogHeader`, `DialogTitle`, and `DialogClose` components from Shadcn UI. You cannot return arbitrary JSX; it must be compatible with the Shadcn Dialog structure.
- This pattern allows you to `await` the alert like a standard async function.

### Confirm Modal (modal.confirm)

The `modal.confirm` function allows you to show a confirmation modal with a title, message, and customizable buttons. It internally uses `modal.custom` and returns a `Promise` that resolves with `true` if confirmed or `false` if cancelled.

<ComponentExampleDisplayer componentName={"modal"} exampleName={"variant-confirm"} client:load/>

#### Signature

```tsx
export type ModalOptions = {
    hasCloseButton?: boolean;
    title?: ReactNode;
    titleIcon?: ReactNode;
    showTitleIcon?: boolean;
}
export type ConfirmModalOptions = {
    message?: ReactNode | ReactNode[];
    confirmButtonContent?: ReactNode;
    cancelButtonContent?: ReactNode;
} & ModalOptions;
modal.confirm: (options: ConfirmModalOptions) => Promise<boolean>;

```
#### Parameters

**`options: ConfirmModalOptions`**
An object with the following optional properties:

- `title?: string` – The title of the confirmation modal. Defaults to `"Confirm"`.
- `message: ReactNode` – The main message content of the modal.
- `cancelButtonContent?: ReactNode` – Custom content for the cancel button. Defaults to an outlined `"Cancel"` button.
- `confirmButtonContent?: ReactNode` – Custom content for the confirm button. Defaults to a filled `"Confirm"` button.
- `hasCloseButton?: boolean` – Whether to show a close button at the top-right corner. Defaults to `true`.

#### Returns

**`Promise<boolean>`**
A promise that resolves with:

- `true` – if the user confirms (clicks the confirm button).
- `false` – if the user cancels or closes the modal.

#### Notes

- The confirm modal is fully controlled by `modal.custom`, but you only need to provide the `ConfirmModalOptions`.
- The modal uses `OpalusDialogContent`, `DialogHeader`, `DialogTitle`, and `DialogClose` components from Shadcn UI. You cannot return arbitrary JSX; it must be compatible with the Dialog structure.
- This pattern allows you to `await` the confirm modal like a standard async function.

### Prompt Modal (modal.prompt)

The `modal.prompt` function allows you to show a modal that prompts the user for input. It displays a message, an input field, and customizable confirm/cancel buttons. It internally uses `modal.custom` and returns a `Promise` that resolves with the entered string or `null` if cancelled.

<ComponentExampleDisplayer componentName={"modal"} exampleName={"variant-prompt"} client:load/>

#### Signature

```ts
export type ModalOptions = {
    hasCloseButton?: boolean;
    title?: ReactNode;
    titleIcon?: ReactNode;
    showTitleIcon?: boolean;
}
export type PromptModalOptions = {
    message?: ReactNode | ReactNode[];
    defaultValue?: string;
    placeholder?: string;
    confirmButtonContent?: ReactNode;
    cancelButtonContent?: ReactNode;
} & ModalOptions;
modal.prompt: (options: PromptModalOptions) => Promise<string | null>;
```

#### Parameters

**`options: PromptModalOptions`**
An object with the following optional properties:

- `title?: string` – The title of the prompt modal. Defaults to `"Prompt"`.
- `titleIcon?: ReactNode` – An optional icon to show next to the title.
- `showTitleIcon?: boolean` – Whether to display the title icon. Defaults to `true`.
- `message?: ReactNode | ReactNode[]` – The main message content of the modal.
- `defaultValue?: string` – The initial value to show in the input field. Defaults to an empty string.
- `placeholder?: string` – Placeholder text for the input field.
- `confirmButtonContent?: ReactNode` – Custom content for the confirm button. Defaults to a filled `"Confirm"` button.
- `cancelButtonContent?: ReactNode` – Custom content for the cancel button. Defaults to an outlined `"Cancel"` button.
- `hasCloseButton?: boolean` – Whether to show a close button at the top-right corner. Defaults to `true`.

#### Returns

**`Promise<string | null>`**
A promise that resolves with:

- The string entered by the user if confirmed.
- `null` if the user cancels or closes the modal.

#### Notes

- The prompt modal is fully controlled by `modal.custom`, but you only need to provide the `PromptModalOptions`.
- The modal uses `OpalusDialogContent`, `DialogHeader`, `DialogTitle`, `DialogClose`, and `Input` components from Shadcn UI. You cannot return arbitrary JSX; it must be compatible with the Dialog structure.
